🎯 Interview Tips

Think Aloud

Narrate your thought process. Interviewers value communication as much as correctness.

Ask Clarifying Questions

Confirm constraints, input range, and edge cases before writing a single line.

Discuss Complexity

Always state time and space complexity. Mention trade-offs between approaches.

Test Your Code

Walk through at least one example manually after writing. Catch bugs before the interviewer does.

Start Brute Force

A correct O(n²) solution is better than an incorrect O(n) attempt. Optimize after.

Code Review Checklist

Off-by-one? Null check? Integer overflow? Duplicate handling? Run this mentally before submitting.
