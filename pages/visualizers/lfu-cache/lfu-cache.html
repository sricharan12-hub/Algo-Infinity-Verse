System Design: LFU Cache (Least Frequently Used)

Status: Cache Initialized | Capacity: 4

Operations

Execution Trace

1. If key exists in Map:
Update node.value
Call internal Get(key) to promote
2. Else (New Key):
If Cache is Full (Size == Cap):
DLL = freqMap[minFreq]
Evict DLL.tail (LRU of Min Freq)
Delete from Map, Size--
Create new Node(freq=1)
Insert into freqMap[1] Head
minFreq = 1, Add to Map, Size++
1. If key NOT in Map:
Return -1 (Cache Miss)
2. Else (Cache Hit):
node = Map[key], f = node.freq
Remove node from freqMap[f]
If freqMap[f] is empty & minFreq == f:
minFreq++
node.freq++
Insert node at freqMap[f+1] Head
Return node.value

Theory

An LFU Cache evicts the element accessed least often. It requires an O(1) Freq Map pointing to multiple Doubly Linked Lists.

  • Promotion Accessing a node increments its frequency, moving it to a higher tier DLL.
  • LRU Tie-Break If the cache is full, it looks at the `minFreq` tier and evicts the Least Recently Used node (Tail) in that specific list.
