🌳 Interval Tree Explained

🌲

BST Ordered by Low Endpoint

Intervals are inserted like a plain BST using their low (start) endpoint as the key. This gives O(log n) insert and a structural property we can exploit for search.

📈

Augmentation: max Field

Each node stores max — the largest high endpoint in its entire subtree. On every insert, max is updated up the ancestor chain. This single extra integer is what enables pruning.

✂️

Pruning With max

During a query [lo, hi]: if a subtree's max < lo, every interval in it ends before our query starts — skip the entire branch. This reduces comparisons from O(n) to O(log n + k).

📅

Real-World Uses

Calendar conflict detection, genome region lookup, compiler register allocation, computational geometry sweep-lines, and database query planners all rely on interval overlap queries.
