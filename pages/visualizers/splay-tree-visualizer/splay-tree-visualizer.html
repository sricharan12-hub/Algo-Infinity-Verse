🌳 The Three Splay Cases

1️⃣

Zig

The accessed node's parent is the root. A single rotation brings the node to the root directly — the base case, used only when the node is one step from the top.

2️⃣

Zig-Zig

The node and its parent are both left children (or both right children) of their respective parents. Rotate the grandparent first, then the parent — same direction, twice.

3️⃣

Zig-Zag

The node and its parent are on opposite sides (one left child, one right child). Rotate the node's parent, then rotate the node again the other direction — a double rotation in different directions.

📊

Amortized O(log n)

Any single splay operation can cost O(n) in the worst case — but Sleator and Tarjan proved that any sequence of m operations costs O(m log n) total, giving an amortized O(log n) per operation, the same asymptotic guarantee as AVL or Red-Black trees, with none of the explicit balance bookkeeping.
