1. Ensure A is smaller: M ≤ N
2. low = 0, high = M
3. While low ≤ high:
4. partA = floor((low + high) / 2)
5. partB = floor((M + N + 1) / 2) - partA
6. Fetch L_A, R_A, L_B, R_B
7. If (L_A ≤ R_B) AND (L_B ≤ R_A):
Partition Found! Calculate Median.
8. Else if (L_A > R_B):
high = partA - 1 (Move Left)
9. Else:
low = partA + 1 (Move Right)

Theory

Instead of merging the arrays, we find a Partition Line that splits both arrays such that the total number of elements on the left equals the total on the right.

  • Part A Binary search cuts the smaller array.
  • Part B The cut for B is mathematically forced: `(M+N+1)/2 - PartA`.
  • Cross Check Valid if `MaxLeftA <= MinRightB` AND `MaxLeftB <= MinRightA`.
