Bridge 01

Recursion → Dynamic Programming

Dynamic Programming is simply Recursion with a memory cache. Learn how to transform exponential time complexities $O(2^N)$ into linear ones $O(N)$ by capturing overlapping subproblems.

Code Evolver (Fibonacci)

Click the tabs to see how the code evolves from pure recursion to Memoization, and finally to Tabulation.

fibonacci_recursive.py Python 
# Pure Recursive Fibonacci
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)
Time: O(2^N) Space: O(N)

Subproblem Recurrence Tree (N=4)

Enable memoization to see duplicate subproblem branches pruned instantly!

Total operations: 15
💡

Without memory, fib(3), fib(2) etc. are calculated repeatedly. Notice the duplicate branches in the tree. This results in exponential operations!

Sorting → Divide & Conquer

Divide and Conquer is a code design philosophy. Learn how the steps of Merge Sort map directly to the three pillars of this paradigm: Divide (Split), Conquer (Base Case Solve), and Combine (Merge).
